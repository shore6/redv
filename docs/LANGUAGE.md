# redv 言語仕様

`.rv` ファイルの文法と、シミュレーションエンジンの挙動(セマンティクス)を定義します。
概要・ビルド方法は [../README.md](../README.md) を参照してください。

---

## 1. 全体構造

- `logic name(input a, output y) { ... }` … レッドストーン回路の定義
- `module name() { ... }` … テストベンチ。`var` 宣言と `sim { ... }` ブロックを書く
- 文末は `;`。ブロックは `{}`。コメントは `//`(行)と `/* */`(範囲)
- ファイル内の全 `module` が **宣言順** に実行される(回路はモジュールごとに独立)

## 2. 宣言子

| 宣言 | 意味 |
|---|---|
| `input` / `output` | ポート。無条件で `reg` 型(= 点) |
| `wire` | **再利用可能な素子列**(端点を持たないテンプレート)。チェーンの中間に展開する |
| `reg` | 点。単体素子を代入する |
| `const reg r = 15b;` | 素子も値も不変。**信号強度の指定が必須**(0–15、範囲外はエラー) |
| `mutable reg r = d;` | 素子は不変、値は回路によって変化 |
| `var` | テスト用の整数変数(C の `int` 相当)。**回路には組み込めない** |

## 3. 素子

| 表記 | 素子 | 動作 |
|---|---|---|
| `d` / `dx` | ダスト(点 / 十字) | 即時伝搬、1 個につき強度 −1 |
| `dn`(例 `d3`) | ダスト n 個 | `ddd...` と同じ |
| `r`, `r1`–`r4` | リピータ | n tick 遅延、出力は 15 に再増幅。1tick パルスも保持 |
| `t` | トーチ | 1 tick 遅延で反転(入力 > 0 → 0、入力 = 0 → 15) |
| `cc` / `cd` | コンパレータ(比較 / 減算) | 1 tick 遅延。サイド入力なしなら **強度をそのまま** 通す(`c` 単体はエラー) |
| `b` | ブロック | 即時、強度は 0 か 15 のみ |

> `r` は `r1` と同じ(遅延 1)。`r5` 以上はエラー。`c` 単体はモード未指定のためエラー(`cc` / `cd` を使う)。

### 3.1 コンパレータのサイド入力

コンパレータは **後ろ(back)入力・横(side)入力・前(out)出力** の 3 端子を持つ。
出力は 1 tick 遅延で、モードにより次のように決まる:

- `cd`(減算): `out = max(0, back − side)`
- `cc`(比較): `out = back >= side ? back : 0`

**サイド入力なし(横未接続 = side 0)なら両モードとも `back` のパススルー** に退化する。
よってワイヤーチェーン内にインラインで置いた `cc` / `cd`(例 `a-d4ccd4-y`)は従来どおり強度中継として働く。

サイド入力を結線するには、コンパレータを **reg に代入して名前付きの点** にし、後ろ・横・前を
別々のワイヤーでつなぐ:

```rv
reg r = cd;          // 減算コンパレータ(cc = 比較)。修飾子は plain のみ・強度指定は不可
a - d3 - r;          // r が終端 → 後ろ(back)入力。減衰も普通に書ける
b - r.side;          // r.side → 横(side)入力
r - y;               // r が始端 → 前(out)出力
```

| 端子 | 書き方 | 役割 |
|---|---|---|
| back | `r`(チェーン終端) | 後ろ入力 |
| side | `r.side`(チェーン終端のみ) | 横入力 |
| out | `r`(チェーン始端) | 前出力 |

- back / side それぞれに複数ワイヤーが合流したら **強度の最大値**(ゲームの左右サイド max と一致)
- `r.side` を **始端(信号源)** に使うのはエラー(横は入力専用)
- `.side` を **コンパレータ / リピーター以外の reg** に付けるのはエラー
- コンパレータ reg に `const` / `mutable` 修飾子・信号強度リテラルを付けるのはエラー

### 3.2 リピーターロック(リピーターのサイド入力)

リピーターも reg に代入して **名前付きの点** にすると、コンパレータと同じく
**後ろ(back)入力・横(side / ロック)入力・前(out)出力** の 3 端子を持つ。
横入力が `> 0` の間、出力は **ロック直前の値で凍結** され、後ろ入力が変化しても
追従しない。横入力が 0 に戻ると通常のリピーターに復帰する(ロックは 1 tick 反応)。

```rv
reg m = r;           // ロック可能リピーター(r / r1-r4。遅延は同じ規則)。修飾子は plain のみ・強度指定は不可
a - m;               // m が終端 → 後ろ(back)入力
lk - m.side;         // m.side → 横(ロック)入力
m - y;               // m が始端 → 前(out)出力
```

| 端子 | 書き方 | 役割 |
|---|---|---|
| back | `m`(チェーン終端) | 後ろ入力 |
| side | `m.side`(チェーン終端のみ) | 横(ロック)入力 |
| out | `m`(チェーン始端) | 前出力 |

- back / side の合流・`.side` の制約はコンパレータ(§3.1)と同じ
- リピーター reg に `const` / `mutable` 修飾子・信号強度リテラルを付けるのはエラー
- チェーン内インラインのリピーター(例 `a-r-b`)は従来どおりロックなし(横入力を持てない)
- **ゲームとの差異**: 本処理系は横入力が `> 0` なら **どんな経路(ダスト等)でもロック**
  する。Minecraft では横がリピーター/コンパレータからのときだけロックする点が異なる

## 4. ワイヤーと点

2 つの reg / ポートを素子列でつなぐには、**チェーン文**(`from -素子...- to ;`)を直接書く。
wire 変数の宣言は不要。

```rv
a-ddr2brdccbr4d3-b;   // a と b を素子列でつなぐ(チャンクは連結して書ける)
a-d-r2-t-b;           // '-' で区切って書いてもよい
a-b;                  // 長さ0のチェーン(直結)
```

- チェーンの先頭と末尾は **宣言済みの reg / ポート**、中間は **素子 or wire 名**
- 1 つの reg に複数のチェーンが合流した場合、**強度の最大値** をとる(ゲーム仕様準拠)
- 端点に `.side` を付けるとコンパレータ / リピーター reg の横入力に結線(§3.1 / §3.2)。中間チャンクには付けられない

### 4.0 wire(再利用可能な素子列)

`wire` は **端点を持たない素子列のテンプレート** で、チェーンの中間チャンクとして展開できる。
同じ wire を複数箇所で使うと、**各箇所で独立した素子列**として展開される(物理的に別物)。

```rv
wire seg;
seg = d2ccd2;             // ダスト2 + コンパレータ + ダスト2 の素子列を定義
a-seg-m;                  // a を seg 経由で m へ
b-seg-m;                  // 同じ seg を b -> m にも再利用(各々別インスタンス)
```

- wire の代入(`w = ...`)は **接続ではなく素子列の定義**。端点(reg/ポート名)や `.side` は書けない
- 単一素子なら `w = r4;` のように書ける。複数素子は `w = d4-cc-d4;` も `w = d4ccd4;` も可
- wire は他の wire を含められる(`wrap = base-r-base;`)。**循環参照はエラー**
- 未定義(宣言のみで未代入)の wire をチェーンで使うのはエラー
- wire の再代入は **警告** を出し、最後の代入が採用される
- wire 自体はチェーンの **端点にはできない**(端点は reg / ポートのみ)

### 4.1 logic の階層化(インスタンス化)

`logic` の本体から別の `logic` を呼び出して入れ子の回路を組める。

```rv
logic and2(input a, input b, output y) {
    reg na, nb, o;
    na = not_gate(a);       // out = callee(args...)
    nb = not_gate(b);
    o  = or_gate(na, nb);
    y  = not_gate(o);       // a & b = !( !a | !b )
}
```

- 形は `out = callee(args...);`。`out` は **このlogic内の reg / 出力ポート**、`args` は
  **reg / 入力ポート名**(wire や素子・数値リテラルは不可)
- `out` には下位 logic の **出力ポートが結線**(減衰なし直結)され、`args` の各ノードが下位の
  **入力ポートを駆動** する。下位インスタンスは sim 変数とは無関係に回路として展開される
- 束縛は **単一出力形のみ**(下位 logic の出力ポートはちょうど 1 個)。多出力 logic は未対応
- 引数の個数は下位 logic の入力ポート数と一致する必要がある
- **再帰インスタンス化**(自己・相互を含む循環呼び出し)は **エラー**

## 5. sim ブロック

```rv
module test() {
    var a, y;
    sim {
        a = 0;
        y = not_gate(a);   // 初回で回路をインスタンス化、変数を入出力に束縛
        #init              // 定常状態まで待つ($time = 0 となる)
        a = 10;
        #1                 // 1 tick 進める
        ?monitor("時刻=%t | a=% | y=%\n", $time, a, y);
    }
}
```

- `v = logic名(変数...)` … 同じ「logic 名 + 引数列」は同一インスタンス。入力変数は継続的に束縛され、
  出力変数は毎 tick 更新される
- `#n` … n tick 進める。`#init` … 更新キューが空(定常状態)になるまで待つ
- `?monitor(...)` … Verilog の `$monitor` 風。sim 開始時に登録され、**すべてのウェイト
  (`#init` / `#n`)完了直後** に自動実行される(sim 内のどこに書いても全時刻が出力される)
- `monitor(fmt, ...)` … その場で出力。`%` / `%d` / `%t` が次の引数を消費、`%2d` のような幅指定可、
  `%%` で `%`
- `wait(n)` … `$time` を進めず、monitor も発火させずに n tick 待つ(発振回路用)
- `v = scan()` … **stdin** から空白/改行区切りの整数を 1 つ読み、変数 `v` に代入する。
  EOF(入力切れ)・非数値は **エラー**。回路へ束縛した変数は読み取り後も 0–15 にクランプされる。
  テストでは `redv foo.rv < input.txt` のように入力を固定すれば決定的に再現できる
- `$time` … `#init` 完了時(未使用なら sim 開始時)を 0 とした経過 tick
- `if` / `while` / `for` … C 準拠(`for (i = 0; i < 4; i = i + 1) { ... }`)

## 6. ディレクティブ

```rv
#define INIT_TIMEOUT 200      // #init のタイムアウト(既定 1000 tick)
#define BURNOUT_LIMIT 8       // トーチ焼き切れ: 窓内トグル回数(既定 8)
#define BURNOUT_WINDOW 30     // 同: 監視窓(既定 30 tick)
#define BURNOUT_COOLDOWN 30   // 同: 強制 OFF 期間(既定 30 tick)
#define MODE element          // 素子レベルモード(logic モードは将来対応)
#include "other.rv"           // ファイル取り込み(stdlogic は未実装で警告)
```

## 7. シミュレーション・セマンティクス(設計判断の確定値)

1. **ティック粒度** — `#1` = 1 レッドストーンティック。ゲームティック素子・1rstick 未満のパルスは
   扱わない。
2. **同一ティック内の評価順序** — 各 tick で「順序素子の出力確定 → 組合せ網(ダスト / ブロック)の
   不動点解決 → 順序素子の入力サンプリング」の 3 相。合流は max をとる単調更新なので不動点は決定的に
   収束し、走査順に結果が依存しない(逐次キュー方式と同じ結果を、順序非依存で保証する実装)。
3. **信号伝搬** — 減衰はティックごとに動的計算。ダスト 1 個につき −1、合流点は最大値。
4. **素子の更新タイミング**
   - ダスト / ブロック: 即時
   - リピータ `rn`: `out(T) = in(T−n) > 0 ? 15 : 0`(1tick パルス保持)。
     ロック付き(横入力あり)は横入力 `> 0` の間 `out(T) = out(T−1)`(凍結)に退化する
   - トーチ: `out(T) = in(T−1) > 0 ? 0 : 15`
   - コンパレータ: `out(T) = f(back(T−1), side(T−1))`。サイド未接続は side = 0 なので
     `out(T) = back(T−1)` のパススルーに退化(`cd`: `max(0, back−side)` / `cc`: `back>=side ? back : 0`)
5. **#init の定常判定** — 全ノード値が不変かつ全順序素子のパイプラインが一様になったら終了。
   `INIT_TIMEOUT` tick を超えるとエラー(発振回路向けに `wait(n)` を用意)。
6. **入力変更の反映** — `a = 10;` は次のティックの先頭で回路に反映(Verilog のノンブロッキング相当)。
   変数は `logic(引数)` で束縛したものだけが回路に影響する。
7. **観測タイミング** — `monitor` / 出力変数は **ティック処理完了後** の値を見る。
8. **同一 reg への複数チェーン合流** — ゲーム仕様通り最大値。
9. **エラー / 警告(厳しめ)**
   - 範囲外の var → input: 0/15 に丸めて **警告**(変数ごとに 1 回)
   - const reg の範囲外強度・再代入: **エラー**
   - 未接続の output ポート: インスタンス化時に **エラー**
   - wire / reg 素子の再代入: **警告**(最終代入を採用)
   - トーチ焼き切れ: 窓内トグル超過で **警告** + 一定期間強制 OFF
   - 名前重複・未宣言変数・引数個数不一致・`c` 単体・`r5` 以上: **エラー**
